# TrustEdge — Predictive Customer Outreach for Connection and Retention

## Problem Statement

This project addresses the problem statement: Predictive Customer Outreach for Connection and Retention. TrustEdge monitors simulated bank customer activity logs and uses machine learning (Random Forest) combined with a dual-LLM Empathy AI (SAGE) to predict churn risk and deliver personalized, calibrated retention outreach in real time.

## How to Run Locally

Step-by-step instructions:

1. Clone the repo: `git clone https://github.com/your-team/trustedge`
2. Install backend dependencies: `cd server && npm install`
3. Install frontend dependencies: `cd ../client && npm install`
4. Generate synthetic data: `cd ../ml && python generate_dataset.py`
5. Setup database: `cd ../server && npx prisma db push && npx prisma db seed`
6. Launch the backend API: `npm run dev`
7. Launch the frontend: `cd ../client && npm run dev`
8. Open browser at: `http://localhost:5173`

## Project Structure

Briefly describe what each folder/file does:

## Dataset

All data is 100% synthetic, generated by our team using `generate_dataset.py`.
It simulates 1,200 bank customers across various roles over 90 days, including:

- Transaction amounts, categories, and salary drops
- App login frequencies and digital transaction ratios
- Account events (SIP cancellations, competitor transfers, EMI misses)
- Anomalous churn patterns injected for ~13.6% of users

No real bank data was used.

## Known Limitations

Be honest — evaluators respect honesty:

- Trained only on synthetic data; would need real transaction data for production use.
- Current system processes batch API calls and is not yet integrated with a real-time data stream (e.g., Kafka).
- The SAGE Empathy AI operates without a RAG pipeline; knowledge is prompt-encoded.
- The Outreach delivery engine simulates messages (Twilio, SendGrid) rather than making live API calls for the POC.
- Used Random Forest instead of Temporal Graph Neural Networks due to compute and scope constraints.
